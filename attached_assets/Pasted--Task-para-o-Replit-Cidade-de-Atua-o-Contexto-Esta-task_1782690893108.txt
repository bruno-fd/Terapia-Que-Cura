# Task para o Replit — Cidade de Atuação

## Contexto

Esta task adiciona o campo de **cidade de atuação** em três pontos do site:
1. Painel do advogado (`painel-perfil.html`) — onde o advogado cadastra suas cidades
2. Página de busca (`advogados.html`) — onde o usuário filtra por cidade
3. Card do advogado na listagem — onde a cidade aparece visivelmente

A fonte de dados é a **API pública do IBGE**, gratuita e sem autenticação.

---

## API do IBGE — como usar

### Buscar municípios por estado (UF)
```
GET https://servicodados.ibge.gov.br/api/v1/localidades/estados/{UF}/municipios
```
Exemplo para SP:
```
https://servicodados.ibge.gov.br/api/v1/localidades/estados/SP/municipios
```

Retorna array JSON. Cada item tem:
```json
{
  "id": 3550308,
  "nome": "São Paulo",
  "microrregiao": {
    "mesorregiao": {
      "UF": {
        "sigla": "SP",
        "nome": "São Paulo"
      }
    }
  }
}
```

Usar apenas `nome` e a sigla da UF. Ignorar os demais campos.

### Estratégia de carregamento

Carregar as cidades sob demanda — somente quando o advogado selecionar um estado. Não carregar todos os 5.570 municípios de uma vez.

Cache local em JavaScript: após carregar as cidades de um estado, armazenar em um objeto para não fazer nova requisição se o estado for selecionado novamente na mesma sessão.

```javascript
const cidadesCache = {};

async function carregarCidades(uf) {
  if (cidadesCache[uf]) return cidadesCache[uf];
  const res = await fetch(
    `https://servicodados.ibge.gov.br/api/v1/localidades/estados/${uf}/municipios`
  );
  const data = await res.json();
  const cidades = data.map(m => m.nome).sort();
  cidadesCache[uf] = cidades;
  return cidades;
}
```

Comportamento do autocomplete:

- Filtrar as cidades do estado selecionado que contenham o texto digitado (case-insensitive, sem acento)
- Máximo 8 sugestões visíveis no dropdown com scroll
- Dropdown com fundo `--color-neutral-0`, `--shadow-md`, `--radius-md`, border `--border-color`
- Cada sugestão: padding 10px 16px, hover em `--color-primary-50`, cursor pointer
- Ao clicar em uma sugestão: adicionar a cidade à lista de cidades selecionadas (ver abaixo) e limpar o campo
- Ao pressionar Enter com texto que não corresponde a nenhuma cidade: ignorar (não permitir cidade fora da base)
- Fechar dropdown ao clicar fora

Função de normalização para busca sem acento:
```javascript
function normalizar(str) {
  return str.normalize('NFD')
    .replace(/[\u0300-\u036f]/g, '')
    .toLowerCase();
}
```

## Instruções técnicas

- A API do IBGE é pública, HTTPS, sem autenticação. Usar `fetch` nativo.
- Sempre tratar erros de rede: se a API falhar, exibir mensagem "Não foi possível carregar as cidades. Tente novamente." com botão de retry.
- Cache em variável JavaScript (objeto literal) — não usar localStorage para o cache de cidades.
- A normalização de acentos deve funcionar para buscas como "sao paulo" encontrar "São Paulo".
- O autocomplete deve funcionar tanto no painel do advogado quanto na busca do usuário — extrair a lógica para uma função reutilizável.
- Não usar nenhuma biblioteca externa. Tudo em JavaScript puro.
- O seletor de estado e o autocomplete de cidade devem ser acessíveis via teclado: Tab para navegar, Enter para selecionar, Esc para fechar dropdown.
- Responsivo obrigatório em todos os pontos.
